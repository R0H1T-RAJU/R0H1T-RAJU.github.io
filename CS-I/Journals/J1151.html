1.1: How does branching enable significantly more complex program behavior?
It enables the use of multiple different pathways or "branches" which allows the code to perform various task based on a single boolean statement.

1.2: Why do you think the term consequent is used to refer to the statements that are executed when the boolean test evaluates to true?
It's referred to as consequent because the statement is executed as a consequence of the boolean statement being true.

2.2: Why is the first jump instruction conditional? What determines whether the jump will be executed or not?
The first jump instruction is conditional because we only want it to run if a certain condition is met. In this instance, the condition that determines whether the jump will be executed is if the value stored in the memory is less than or equal to 4.

2.2: What is the purpose of the jump instruction after the consequent?
So that the program will skip over the "alternative" function and jump directly to the "afterAlternate" function.

2.3: Why is there no need for a corresponding jump instruction after the alternative?
There is no need for a corresponding jump instruction because there nothing it needs to skip over. Since there are no other functions between the "alternative" and "afterAlternative" functions, it can execute them in sequence without needing to jump.
